# Getting Started — First Week

The system only compounds if you feed it. Here's the minimum to get value fast.

## Day 1 — Stand it up
- [ ] Install [Obsidian](https://obsidian.md) and **open the `vault/` folder** as a vault.
- [ ] Skim `CLAUDE.md` once so you know how filing decisions get made.
- [ ] Look at the two seed notes in `vault/20-wiki/` to see the shape of a clean note.

## Day 2 — Capture everything (don't organize)
- [ ] Every customer text/email/call → paste into `vault/00-inbox/` as a new note.
- [ ] Every measurement, quote, install gotcha → into `00-inbox/`.
- [ ] Don't tidy. Raw is messy by design.

## Day 3 — Run the processor
- [ ] From `blinds-company/`, open Claude Code and say **"process the inbox."**
- [ ] Review what it filed in `vault/20-wiki/` and check `agents/logs/processing-log.md`.
- [ ] Fix any `> NEEDS REVIEW` callouts it left.

## Day 4–5 — Wire up a workflow
- [ ] Pick one repeatable pain (most people: quote follow-ups) and run
      `agents/workflows/quote-follow-up.md`.
- [ ] Read the run log. Tweak the workflow spec so the next run is sharper.

## Ongoing rhythm
- **Daily:** dump raw into `00-inbox`. 30 seconds, no thinking.
- **Daily/whenever:** "process the inbox" so the wiki stays current.
- **Weekly:** run the operator workflows (follow-ups, supplier-price watch).

## What "good" looks like after a month
- `00-inbox` is usually empty.
- Every active customer has exactly one note with a dated timeline.
- You answer "what did we quote the Garcias and when?" by opening one note, not digging.
